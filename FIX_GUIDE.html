Your EOD reports cannot be updated because the database is missing an UPDATE policy. Follow the steps below to fix this in 5 minutes.

1
Open Supabase SQL Editor

Go to your Supabase Dashboard:

  1. Log in to supabase.com
  2. Select your LIGI Reporting project
  3. Click "SQL Editor" in the left sidebar
  4. Click "New query" button
üì∏ Supabase Dashboard ‚Üí SQL Editor ‚Üí New Query
2
Run the Fix SQL

Copy and paste this SQL code into the editor:

-- Allow branch managers to update their EOD reports CREATE POLICY "Branch managers can update their EOD reports" ON eod_reports FOR UPDATE USING ( EXISTS ( SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'branch-manager' AND profiles.branch_id = eod_reports.branch_id ) ) WITH CHECK ( EXISTS ( SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'branch-manager' AND profiles.branch_id = eod_reports.branch_id ) ); -- Allow business owners to update EOD reports CREATE POLICY "Business owners can update all EOD reports" ON eod_reports FOR UPDATE USING ( EXISTS ( SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'business-owner' ) ) WITH CHECK ( EXISTS ( SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'business-owner' ) );
